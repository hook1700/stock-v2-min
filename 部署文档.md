## 环境要求

### 必需软件

| 软件 | 最低版本 | 推荐版本 | 下载地址 |
|------|----------|----------|----------|
| Python | 3.10+ | 3.11 | https://www.python.org/downloads/ |
| Node.js | 18+ | 20 LTS | https://nodejs.org/ |
| Git | 2.30+ | 最新版 | https://git-scm.com/ |

### 系统要求

- **操作系统**：Windows 10/11（推荐）、macOS、Linux
- **内存**：4GB以上（推荐8GB）
- **磁盘空间**：约500MB（含依赖包）
- **网络**：需要联网获取股票数据（AKShare接口）

### 验证环境

打开命令行（CMD/PowerShell/Terminal），执行以下命令确认环境：

```bash
python --version    # 应显示 Python 3.10.x 或更高
node --version      # 应显示 v18.x.x 或更高
npm --version       # 应显示 9.x.x 或更高
```

## 快速启动

### Windows 用户（推荐）

**双击 `start.bat` 即可一键启动**，脚本会自动完成：
1. 检查 Python 和 Node.js 环境
2. 创建 Python 虚拟环境并安装后端依赖
3. 安装前端 npm 依赖
4. 启动后端服务（端口 8000）
5. 启动前端服务（端口 5173）

启动成功后，浏览器访问：**http://localhost:5173**

## 手动部署步骤

### 第一步：获取项目代码

```bash
# 进入项目目录
cd E:\myproject\stock-v2-min
```

### 第二步：部署后端

```bash
# 1. 进入后端目录
cd backend

# 2. 创建 Python 虚拟环境
python -m venv venv

# 3. 激活虚拟环境
# Windows CMD:
venv\Scripts\activate.bat
# Windows PowerShell:
venv\Scripts\Activate.ps1
# macOS/Linux:
source venv/bin/activate

# 4. 安装依赖（使用清华镜像加速）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 5. 启动后端服务（注意：需要回到项目根目录执行，以 backend 作为包导入）
cd ..
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

看到以下输出表示后端启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
正在启动股票选股策略系统...
系统启动完成
定时任务已启动：每个交易日 15:30 执行
```

### 第三步：部署前端

打开**新的命令行窗口**：

```bash
# 1. 进入前端目录
cd E:\myproject\stock-v2-min\frontend

# 2. 安装依赖（使用淘宝镜像加速）
npm install --registry=https://registry.npmmirror.com

# 3. 启动前端开发服务器
npm run dev
```

看到以下输出表示前端启动成功：
```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

| http://localhost:5173 | 前端界面（主界面） |

## 系统配置

### 后端配置

配置文件位于 `backend/config.py`，可通过 `.env` 文件覆盖默认值。

#### 创建 .env 文件（可选）

在 `backend/` 目录下创建 `.env` 文件：

```env
# 定时任务时间（默认15:30）
DAILY_RUN_HOUR=15
DAILY_RUN_MINUTE=30

# 策略参数调整
VOLUME_RATIO_THRESHOLD=1.5
DIVIDEND_YIELD_MIN=0.04
MIN_MARKET_CAP=500000000

# 每个策略推荐股票数
TOP_PICKS=3
```

#### 关键配置说明

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `DAILY_RUN_HOUR` | 15 | 定时任务执行小时 |
| `DAILY_RUN_MINUTE` | 30 | 定时任务执行分钟 |
| `TOP_PICKS` | 3 | 每个策略推荐股票数量 |
| `MIN_MARKET_CAP` | 5亿 | 最低市值过滤（元） |
| `VOLUME_RATIO_THRESHOLD` | 1.5 | 杯柄策略突破量比阈值 |
| `DIVIDEND_YIELD_MIN` | 0.04 | 高股息策略最低股息率 |
| `DEBT_RATIO_MAX` | 0.60 | 最高资产负债率 |
| `BOTTOM_DECLINE_MONTHS` | 3 | 底部策略前期下跌月数 |

### 前端配置

前端通过 `vite.config.js` 中的 proxy 配置连接后端：

```javascript
// 默认代理到 localhost:8000
proxy: {
  '/api': {
    target: 'http://localhost:8000',
    changeOrigin: true,
  },
}
```

如需修改后端地址，编辑此文件即可。

## 使用指南

### 首次使用

1. **启动系统**后，打开 http://localhost:5173
2. 点击右上角 **「立即执行」** 按钮，手动触发全部策略
3. 等待执行完成（首次约需5-15分钟，取决于网络速度）
4. 刷新页面查看推荐结果

### 日常使用

系统会在每个**交易日 15:30**（收盘后）自动执行全部策略，无需手动操作。

#### 功能页面

| 页面 | 路径 | 说明 |
|------|------|------|
| 策略总览 | `/` | 查看4个策略的最新推荐（每个3只股票） |
| 策略详情 | `/strategy/1~4` | 单个策略的详细推荐，含买卖点 |
| 板块轮动 | `/sectors` | 申万板块机会/风险分析及推荐股票 |
| 个股K线 | `/stock/代码` | 个股K线图+均线 |
| 历史推荐 | `/history` | 所有历史推荐记录 |

#### 手动触发策略

- **全部策略**：点击右上角「立即执行」
- **单个策略**：进入策略详情页 → 点击「重新执行此策略」
- **API触发**：`POST http://localhost:8000/api/system/run-all`

### API 接口

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/strategies` | 获取所有策略概览 |
| GET | `/api/strategies/{id}` | 获取策略详情 |
| GET | `/api/strategies/{id}/history` | 策略历史推荐 |
| POST | `/api/strategies/{id}/run` | 触发单个策略 |
| GET | `/api/sectors/rotation` | 板块轮动分析 |
| GET | `/api/stocks/{code}/kline?days=180` | 股票K线数据 |
| GET | `/api/system/status` | 系统运行状态 |
| POST | `/api/system/run-all` | 触发全部策略 |

## 常见问题

### Q1: pip install 报错超时或下载失败

**原因**：默认PyPI源在国外，网络不稳定

**解决**：使用国内镜像源
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

或永久配置镜像：
```bash
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q2: npm install 缓慢或失败

**解决**：使用淘宝镜像
```bash
npm install --registry=https://registry.npmmirror.com
```

或永久配置：
```bash
npm config set registry https://registry.npmmirror.com
```

### Q3: 后端启动报 "ModuleNotFoundError"

**原因**：虚拟环境未激活或依赖未安装

**解决**：
```bash
cd backend
# 确保虚拟环境已激活
venv\Scripts\activate.bat
# 重新安装依赖
pip install -r requirements.txt
```

### Q4: 前端页面显示空白/接口404

**原因**：后端服务未启动或端口冲突

**解决**：
1. 确认后端在8000端口运行：访问 http://localhost:8000/docs
2. 如端口被占用，修改后端启动命令：`--port 8001`
3. 同步修改前端 `vite.config.js` 中的 proxy target

### Q5: 执行策略后无推荐结果

**原因**：
- 非交易日（周末/节假日）系统会跳过执行
- 网络问题导致AKShare数据获取失败
- 当日市场无符合条件的股票

**解决**：
1. 查看后端控制台日志确认错误原因
2. 确认网络可以访问 AKShare 数据接口
3. 交易日15:30后再执行

### Q6: AKShare 接口报错 "频率限制"

**原因**：短时间内请求过多触发限制

**解决**：系统已内置重试机制和日级缓存，正常使用不会触发。若频繁手动触发，请间隔5分钟以上。

### Q7: 数据库文件在哪里？如何重置？

**位置**：`backend/data/stock_v2.db`

**重置数据库**：
```bash
# 删除数据库文件，重启后端会自动重建
del backend\data\stock_v2.db
# 清除缓存
del backend\data\cache\*.pkl
```

### Q8: 如何修改定时执行时间？

编辑 `backend/.env` 文件（不存在则创建）：
```env
DAILY_RUN_HOUR=16
DAILY_RUN_MINUTE=0
```
重启后端服务生效。

### Q9: PowerShell 无法执行 activate 脚本

**原因**：PowerShell执行策略限制

**解决**：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q10: 如何在后台持续运行？

**Windows**：使用 `start.bat` 启动，关闭CMD窗口不影响服务

**Linux/macOS**：
```bash
# 使用 nohup 后台运行（在项目根目录执行）
nohup python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 &

cd frontend
nohup npm run dev &
```

或使用 `pm2`（推荐）：
```bash
npm install -g pm2
pm2 start "uvicorn backend.main:app --port 8000" --name stock-backend
pm2 start "npm run dev" --name stock-frontend
```

## 技术栈总览

| 层级 | 技术 | 用途 |
|------|------|------|
| 后端框架 | FastAPI | 高性能异步API服务 |
| 数据获取 | AKShare | A股数据（免费，无需Token） |
| 数据库 | SQLite + SQLAlchemy | 零配置本地持久化 |
| 定时任务 | APScheduler | 每日15:30自动执行 |
| 前端框架 | Vue 3 + Vite | 响应式单页应用 |
| UI组件库 | Element Plus | 中文友好的企业级组件 |
| 图表库 | Apache ECharts | K线图、热力图 |
| 状态管理 | Pinia | Vue3官方推荐 |
| HTTP客户端 | Axios | API请求 |
