## 服务器选购

### 推荐配置

| 配置项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| CPU | 2核 | 2核+ |
| 内存 | 2GB | 4GB |
| 系统盘 | 40GB SSD | 50GB SSD |
| 带宽 | 1Mbps | 3Mbps+ |
| 系统 | CentOS 7+ | Ubuntu 22.04 LTS |
| 地域 | 任意 | 上海/广州（靠近数据源） |

### 腾讯云购买步骤

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 选择 **云服务器 CVM** → **新建实例**
3. 镜像选择：**Ubuntu Server 22.04 LTS 64位**
4. 实例规格：**标准型 S5.MEDIUM4**（2核4G）
5. 网络：选择默认VPC，分配公网IP，带宽选按量计费或固定带宽
6. 安全组：放行 80、443、22 端口
7. 设置密码或SSH密钥
8. 购买并等待实例启动

## Nginx 反向代理配置

### 创建站点配置

```bash
cat > /etc/nginx/sites-available/stock-strategy << 'EOF'
server {
    listen 80;
    server_name your_domain.com;  # 替换为你的域名或服务器IP

    # 前端静态文件
    root /var/www/stock-strategy;
    index index.html;

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1024;

    # 前端路由（Vue Router history模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API反向代理到FastAPI后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;  # 策略执行可能耗时较长
        proxy_connect_timeout 60s;
    }

    # API文档（可选，生产环境建议关闭）
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /redoc {
        proxy_pass http://127.0.0.1:8000/redoc;
        proxy_set_header Host $host;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:8000/openapi.json;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
EOF
```

### 启用站点

```bash
# 创建符号链接启用站点
ln -sf /etc/nginx/sites-available/stock-strategy /etc/nginx/sites-enabled/

# 移除默认站点
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重载Nginx
systemctl reload nginx
```

```bash
certbot --nginx -d stock.yourdomain.com

certbot renew --dry-run
```

### 3. HTTPS Nginx 配置（手动方式）

```bash
cat > /etc/nginx/sites-available/stock-strategy << 'EOF'
# HTTP重定向到HTTPS
server {
    listen 80;
    server_name stock.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS配置
server {
    listen 443 ssl http2;
    server_name stock.yourdomain.com;

    # SSL证书路径（根据实际修改）
    ssl_certificate /etc/nginx/ssl/stock.yourdomain.com.pem;
    ssl_certificate_key /etc/nginx/ssl/stock.yourdomain.com.key;

    # SSL优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;

    # 其余配置同HTTP版本...
    root /var/www/stock-strategy;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
EOF

nginx -t && systemctl reload nginx
```

## 防火墙与安全组

### 腾讯云安全组规则

在控制台 → 安全组 → 添加以下入站规则：

| 类型 | 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|------|
| 入站 | TCP | 22 | 0.0.0.0/0 或指定IP | SSH登录 |
| 入站 | TCP | 80 | 0.0.0.0/0 | HTTP |
| 入站 | TCP | 443 | 0.0.0.0/0 | HTTPS |

> **注意**：不要开放 8000 端口到公网，后端只通过 Nginx 内部转发访问

### 服务器防火墙（UFW）

```bash
# 启用UFW
ufw allow ssh
ufw allow 'Nginx Full'
ufw enable

# 查看状态
ufw status
```

### 安全建议

1. **禁止root SSH登录**，创建普通用户
```bash
adduser deploy
usermod -aG sudo deploy
# 编辑 /etc/ssh/sshd_config 设置 PermitRootLogin no
```

2. **使用SSH密钥登录**，禁用密码登录
3. **生产环境关闭API文档**，编辑 `backend/main.py`：
```python
app = FastAPI(docs_url=None, redoc_url=None)  # 关闭文档
```

## 数据源问题处理（重要）

### 问题现象

后端启动后调 `/api/system/run-all` 时，日志反复出现：

```
get_stock_pool 第N次失败：('Connection aborted.', RemoteDisconnected('Remote end closed connection without response'))
get_stock_pool 第N次失败：HTTPError 403 ...
```

或调用诊断接口 `GET /api/system/datasource-check` 时，`eastmoney_clist` / `sina_hq` 全部 `ok=false`。

### 根本原因

**腾讯云、阿里云等公有云的服务器 IP 段（机房 IP）被东方财富、新浪财经的反爬规则识别并拉黑**，连接在 TLS/HTTP 层就被服务端直接 RST 或 403。

这是国内云服务器跑 AKShare 等行情爬虫的共性问题，**和代码、UA、Referer 都无关**——同样的代码在家用宽带能跑通，在云服务器上跑不通。

### 诊断方法

部署后第一时间调用诊断接口：

```bash
curl -sS http://127.0.0.1:8000/api/system/datasource-check | python3 -m json.tool
```

**关键字段判读**：

| `server_ip_info.country` | 三个 `checks` 的 `ok` 状态 | 结论 |
|---|---|---|
| `CN` | 三个都 `true` | ✅ 数据源正常，可直接使用 |
| `CN` | 三个都 `false` | ⚠️ 机房 IP 被反爬拉黑（最常见，使用方案 A） |
| 非 `CN` | 全部 `false` | ❌ 服务器在境外，必须换大陆地域（方案 C） |

### 解决方案 A：本地拉数据 + 同步缓存到云服务器（推荐 ⭐）

**思路**：让本地家用宽带电脑（IP 不会被封）每天定时拉数据，把缓存文件 `scp` 上传到云服务器；云端策略代码只读缓存，不调 AKShare。

**架构**：

```
[本地家用宽带电脑]                          [腾讯云服务器]
   ↓ 任务计划 (15:30)                        ↑
   1. 调 AKShare 拉数据                      |
   2. 写 backend/data/cache/*.pkl            |
   3. scp 上传 ────────────────────────────→ FastAPI 读缓存
                                             跑策略 / 提供 API
```

代码层面**完全不需要修改**——`DataService` 已实现"先读缓存命中就不调 AKShare"的逻辑，只要 `data/cache/` 有文件，云端就直接用。

#### 步骤 1：本地准备 Python 环境

```powershell
# Windows PowerShell
cd E:\myproject\stock-v2-min\backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
deactivate
```

#### 步骤 2：确认 SSH 私钥可用

项目根目录已有 SSH 私钥 `way`（如果没有，从腾讯云控制台下载或自行创建）。

测试连通：

```powershell
cd E:\myproject\stock-v2-min
ssh -i .\way way@175.178.172.130 "echo OK && hostname"
```

如果用户名不是 `way`（比如是 `root`），运行脚本时设置环境变量覆盖即可。

#### 步骤 3：执行本地拉数据脚本

仓库已提供 `scripts/fetch_and_sync.py`，作用：
- 调 AKShare 拉取实时行情、股票池、K线、申万板块数据；
- 写入 `backend/data/cache/*.pkl`；
- 通过 `scp` 自动上传到云服务器对应目录。

```powershell
cd E:\myproject\stock-v2-min

# 默认拉前 500 只股票的 K 线（约 3-5 分钟）
python scripts\fetch_and_sync.py

# 或双击运行批处理
scripts\fetch_and_sync.bat
```

**可选环境变量**：

| 变量 | 默认值 | 说明 |
|---|---|---|
| `REMOTE_USER` | `way` | SSH 用户名 |
| `REMOTE_HOST` | `175.178.172.130` | 服务器公网 IP |
| `REMOTE_DIR` | `/home/way/stock-v2-min` | 服务器项目根目录 |
| `SSH_KEY` | `./way` | 本地私钥路径 |
| `TOP_N_KLINE` | `500` | 拉取 K 线的股票数（0=全部，约 5000+ 只，20-30 分钟） |

示例：

```powershell
$env:REMOTE_USER="root"
$env:TOP_N_KLINE="0"
python scripts\fetch_and_sync.py
```

#### 步骤 4：服务器端验证

```bash
# 1. 确认缓存文件已同步
ls -lh /home/way/stock-v2-min/backend/data/cache/

# 2. 触发策略执行（此时不再调 AKShare）
curl -X POST http://127.0.0.1:8000/api/system/run-all

# 3. 查看日志确认走了缓存
tail -f /var/log/stock-strategy/backend.log
# 不应再出现 "正在获取A股股票池..." 的网络请求日志
```

#### 步骤 5：设置每日自动运行（Windows 任务计划）

打开「**任务计划程序**」→「**创建基本任务**」：

| 配置项 | 值 |
|---|---|
| 名称 | 股票数据每日同步 |
| 触发器 | 每天 15:30（A股收盘后） |
| 操作 | 启动程序：`E:\myproject\stock-v2-min\scripts\fetch_and_sync.bat` |
| 条件 | 勾选「只有在网络连接可用时才启动」 |
| 设置 | 勾选「如果错过运行时间，启动任务」 |

> **注意**：本地电脑需要在工作日 15:30 后保持开机一段时间，让脚本能完整跑完并上传。

### 解决方案 B：使用付费数据源 Tushare Pro

如果本地电脑无法保证每日开机，可改用 Tushare Pro（云服务器可直连）：

1. 注册并购买积分：https://tushare.pro/ （约 200 元/年）
2. 获取 token；
3. 改造 `data_service.py`，用 `tushare` 包替换 `akshare` 的关键调用：
   ```python
   import tushare as ts
   ts.set_token("YOUR_TOKEN")
   pro = ts.pro_api()
   df = pro.daily(ts_code=code, start_date=..., end_date=...)
   ```

### 解决方案 C：换服务器到大陆地域 + 改用部分接口

仅适用于**服务器本身就在境外**的情况（诊断接口显示 `country` 不是 `CN`）。

把服务器迁到腾讯云上海/广州/北京机房后，**重试方案 A 的诊断接口**——如果还是 `eastmoney_clist=false`，说明仍然是机房 IP 反爬，依然需要走方案 A 或 B。

## 自动化部署脚本

### 一键部署脚本

在服务器上创建 `/opt/stock-strategy/deploy.sh`：

```bash
cat > /opt/stock-strategy/deploy.sh << 'DEPLOY_EOF'
#!/bin/bash
set -e

echo "========================================="
echo "  股票选股策略系统 - 自动部署"
echo "========================================="

PROJECT_DIR="/opt/stock-strategy"
FRONTEND_DIR="$PROJECT_DIR/frontend"
BACKEND_DIR="$PROJECT_DIR/backend"
WEB_DIR="/var/www/stock-strategy"

cd $PROJECT_DIR

# 1. 拉取最新代码（如果使用Git）
# echo "[1/5] 拉取最新代码..."
# git pull origin main

# 2. 后端依赖更新
echo "[2/5] 更新后端依赖..."
cd $BACKEND_DIR
source venv/bin/activate
pip install -r requirements.txt -q -i https://pypi.tuna.tsinghua.edu.cn/simple

# 3. 前端构建
echo "[3/5] 构建前端..."
cd $FRONTEND_DIR
npm install --registry=https://registry.npmmirror.com --silent
npm run build

# 4. 部署前端文件
echo "[4/5] 部署前端静态文件..."
rm -rf $WEB_DIR/*
cp -r $FRONTEND_DIR/dist/* $WEB_DIR/

# 5. 重启后端服务
echo "[5/5] 重启后端服务..."
supervisorctl restart stock-backend

# 等待启动
sleep 3
supervisorctl status stock-backend

echo ""
echo "========================================="
echo "  部署完成！"
echo "========================================="
echo "  访问地址: http://$(curl -s ifconfig.me)"
echo "========================================="
```

## 常见问题

### Q1: 服务器内存不足导致进程被kill

**解决**：添加 Swap 分区
```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

### Q2: AKShare 在服务器上无法获取数据（RemoteDisconnected / 403）

**典型报错**：
```
get_stock_pool 调用失败：('Connection aborted.', RemoteDisconnected('Remote end closed connection without response'))
```

**原因**：

1. 服务器在境外，AKShare 调用的东方财富 / 新浪财经接口仅对大陆 IP 开放；
2. **服务器在大陆，但腾讯云、阿里云的机房 IP 段被反爬封禁**（最常见）。

**快速诊断**：调用本项目内置的诊断接口

```bash
curl -sS http://127.0.0.1:8000/api/system/datasource-check | python3 -m json.tool
```

**解决方案**：详见上文「[数据源问题处理（重要）](#数据源问题处理重要)」章节，推荐使用「**本地拉数据 + 同步缓存到云服务器**」方案。

### Q3: npm run build 内存不足

**解决**：增加 Node.js 内存限制
```bash
export NODE_OPTIONS="--max-old-space-size=2048"
npm run build
```

### Q4: Supervisor 重启后服务不自动启动

**解决**：
```bash
systemctl enable supervisor
# 确认配置中 autostart=true
```

### Q5: 如何查看定时任务是否正常执行

```bash
# 查看后端日志中的定时任务记录
grep "每日选股分析" /var/log/stock-strategy/backend.log | tail -5

# 或通过API查询
curl -s http://127.0.0.1:8000/api/system/status
```

### Q6: 需要更新代码怎么操作

```bash
# 方式1：Git pull + 重新部署
cd /opt/stock-strategy
git pull
./deploy.sh

# 方式2：重新上传后执行部署脚本

scp -r ./frontend/* root@server:/opt/stock-strategy/frontend/

```
