## 服务器选购

### 推荐配置

| 配置项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| CPU | 2核 | 2核+ |
| 内存 | 2GB | 4GB |
| 系统盘 | 40GB SSD | 50GB SSD |
| 带宽 | 1Mbps | 3Mbps+ |
| 系统 | CentOS 7+ | Ubuntu 22.04 LTS |
| 地域 | 任意 | 上海/广州（靠近数据源） |

### 腾讯云购买步骤

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 选择 **云服务器 CVM** → **新建实例**
3. 镜像选择：**Ubuntu Server 22.04 LTS 64位**
4. 实例规格：**标准型 S5.MEDIUM4**（2核4G）
5. 网络：选择默认VPC，分配公网IP，带宽选按量计费或固定带宽
6. 安全组：放行 80、443、22 端口
7. 设置密码或SSH密钥
8. 购买并等待实例启动

## 项目部署

### 1. 创建应用目录

```bash
mkdir -p /opt/stock-strategy
cd /opt/stock-strategy
```

### 2. 上传代码

**方式一：Git 拉取（推荐）**
```bash
# 如果代码在Git仓库
git clone https://your-repo-url.git .
```

**方式二：SCP 上传**
```bash
# 在本地电脑执行（Windows用PowerShell或Git Bash）
scp -r E:\myproject\stock-v2-min\* root@your_server_ip:/opt/stock-strategy/
```

**方式三：腾讯云 COS 中转**
```bash
# 先上传到COS，再从服务器拉取
apt install -y unzip
wget https://your-cos-bucket.cos.ap-shanghai.myqcloud.com/stock-v2-min.zip
unzip stock-v2-min.zip -d /opt/stock-strategy/
```

### 3. 部署后端

```bash
cd /opt/stock-strategy/backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖（使用清华镜像）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 创建数据目录
mkdir -p data/cache

# 测试启动（验证无报错，需回到项目根目录，以 backend 作为包导入）
cd /opt/stock-strategy
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
# 看到 "Uvicorn running on http://0.0.0.0:8000" 表示成功
# Ctrl+C 退出
```

### 4. 构建前端

```bash
cd /opt/stock-strategy/frontend

# 安装依赖（使用淘宝镜像）
npm install --registry=https://registry.npmmirror.com

# 修改API地址为生产环境
# 创建生产环境配置
cat > .env.production << 'EOF'
VITE_API_BASE=/api
EOF

# 构建生产版本
npm run build

# 构建产物在 dist/ 目录
ls dist/
```

### 5. 部署静态文件

```bash
# 将前端构建产物复制到 Nginx 目录
mkdir -p /var/www/stock-strategy
cp -r /opt/stock-strategy/frontend/dist/* /var/www/stock-strategy/
```

## Nginx 反向代理配置

### 创建站点配置

```bash
cat > /etc/nginx/sites-available/stock-strategy << 'EOF'
server {
    listen 80;
    server_name your_domain.com;  # 替换为你的域名或服务器IP

    # 前端静态文件
    root /var/www/stock-strategy;
    index index.html;

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
    gzip_min_length 1024;

    # 前端路由（Vue Router history模式）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API反向代理到FastAPI后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;  # 策略执行可能耗时较长
        proxy_connect_timeout 60s;
    }

    # API文档（可选，生产环境建议关闭）
    location /docs {
        proxy_pass http://127.0.0.1:8000/docs;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /redoc {
        proxy_pass http://127.0.0.1:8000/redoc;
        proxy_set_header Host $host;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:8000/openapi.json;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
EOF
```

### 启用站点

```bash
# 创建符号链接启用站点
ln -sf /etc/nginx/sites-available/stock-strategy /etc/nginx/sites-enabled/

# 移除默认站点
rm -f /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重载Nginx
systemctl reload nginx
```

## 进程管理（Supervisor）

### 创建后端进程配置

```bash
cat > /etc/supervisor/conf.d/stock-backend.conf << 'EOF'
[program:stock-backend]
command=/opt/stock-strategy/backend/venv/bin/uvicorn backend.main:app --host 127.0.0.1 --port 8000 --workers 2
directory=/opt/stock-strategy
user=root
autostart=true
autorestart=true
startsecs=10
stopwaitsecs=30
redirect_stderr=true
stdout_logfile=/var/log/stock-strategy/backend.log
stdout_logfile_maxbytes=50MB
stdout_logfile_backups=5
environment=PYTHONPATH="/opt/stock-strategy"
EOF
```

### 创建日志目录

```bash
mkdir -p /var/log/stock-strategy
```

### 启动服务

```bash
# 重新读取配置
supervisorctl reread
supervisorctl update

# 启动后端
supervisorctl start stock-backend

# 查看状态
supervisorctl status
```

### 常用 Supervisor 命令

```bash
supervisorctl status              # 查看所有进程状态
supervisorctl start stock-backend # 启动
supervisorctl stop stock-backend  # 停止
supervisorctl restart stock-backend # 重启
supervisorctl tail stock-backend  # 查看日志
supervisorctl tail -f stock-backend # 实时日志
```

## 域名与HTTPS配置

### 1. 域名解析

在腾讯云 **DNS解析** 控制台：
1. 添加域名（如 stock.yourdomain.com）
2. 添加 A 记录指向服务器公网IP

### 2. 申请免费SSL证书

**方式一：腾讯云免费证书**
1. 控制台 → SSL证书 → 申请免费证书
2. 选择域名验证方式
3. 下载证书文件（Nginx格式）

**方式二：Let's Encrypt（推荐）**
```bash
# 安装 Certbot
apt install -y certbot python3-certbot-nginx

# 自动申请并配置
certbot --nginx -d stock.yourdomain.com

# 自动续期（已自动配置cron）
certbot renew --dry-run
```

### 3. HTTPS Nginx 配置（手动方式）

```bash
cat > /etc/nginx/sites-available/stock-strategy << 'EOF'
# HTTP重定向到HTTPS
server {
    listen 80;
    server_name stock.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS配置
server {
    listen 443 ssl http2;
    server_name stock.yourdomain.com;

    # SSL证书路径（根据实际修改）
    ssl_certificate /etc/nginx/ssl/stock.yourdomain.com.pem;
    ssl_certificate_key /etc/nginx/ssl/stock.yourdomain.com.key;

    # SSL优化
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;

    # 其余配置同HTTP版本...
    root /var/www/stock-strategy;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
EOF

nginx -t && systemctl reload nginx
```

## 防火墙与安全组

### 腾讯云安全组规则

在控制台 → 安全组 → 添加以下入站规则：

| 类型 | 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|------|
| 入站 | TCP | 22 | 0.0.0.0/0 或指定IP | SSH登录 |
| 入站 | TCP | 80 | 0.0.0.0/0 | HTTP |
| 入站 | TCP | 443 | 0.0.0.0/0 | HTTPS |

> **注意**：不要开放 8000 端口到公网，后端只通过 Nginx 内部转发访问

### 服务器防火墙（UFW）

```bash
# 启用UFW
ufw allow ssh
ufw allow 'Nginx Full'
ufw enable

# 查看状态
ufw status
```

### 安全建议

1. **禁止root SSH登录**，创建普通用户
```bash
adduser deploy
usermod -aG sudo deploy
# 编辑 /etc/ssh/sshd_config 设置 PermitRootLogin no
```

2. **使用SSH密钥登录**，禁用密码登录
3. **生产环境关闭API文档**，编辑 `backend/main.py`：
```python
app = FastAPI(docs_url=None, redoc_url=None)  # 关闭文档
```

## 自动化部署脚本

### 一键部署脚本

在服务器上创建 `/opt/stock-strategy/deploy.sh`：

```bash
cat > /opt/stock-strategy/deploy.sh << 'DEPLOY_EOF'
#!/bin/bash
set -e

echo "========================================="
echo "  股票选股策略系统 - 自动部署"
echo "========================================="

PROJECT_DIR="/opt/stock-strategy"
FRONTEND_DIR="$PROJECT_DIR/frontend"
BACKEND_DIR="$PROJECT_DIR/backend"
WEB_DIR="/var/www/stock-strategy"

cd $PROJECT_DIR

# 1. 拉取最新代码（如果使用Git）
# echo "[1/5] 拉取最新代码..."
# git pull origin main

# 2. 后端依赖更新
echo "[2/5] 更新后端依赖..."
cd $BACKEND_DIR
source venv/bin/activate
pip install -r requirements.txt -q -i https://pypi.tuna.tsinghua.edu.cn/simple

# 3. 前端构建
echo "[3/5] 构建前端..."
cd $FRONTEND_DIR
npm install --registry=https://registry.npmmirror.com --silent
npm run build

# 4. 部署前端文件
echo "[4/5] 部署前端静态文件..."
rm -rf $WEB_DIR/*
cp -r $FRONTEND_DIR/dist/* $WEB_DIR/

# 5. 重启后端服务
echo "[5/5] 重启后端服务..."
supervisorctl restart stock-backend

# 等待启动
sleep 3
supervisorctl status stock-backend

echo ""
echo "========================================="
echo "  部署完成！"
echo "========================================="
echo "  访问地址: http://$(curl -s ifconfig.me)"
echo "========================================="
DEPLOY_EOF

chmod +x /opt/stock-strategy/deploy.sh
```

### 执行部署

```bash
/opt/stock-strategy/deploy.sh
```

## 常见问题

### Q1: 服务器内存不足导致进程被kill

**解决**：添加 Swap 分区
```bash
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
echo '/swapfile none swap sw 0 0' >> /etc/fstab
```

### Q2: AKShare 在服务器上无法获取数据

**原因**：部分AKShare接口需要大陆网络环境

**解决**：确保服务器在中国大陆地域（上海/广州/北京）

### Q3: npm run build 内存不足

**解决**：增加 Node.js 内存限制
```bash
export NODE_OPTIONS="--max-old-space-size=2048"
npm run build
```

### Q4: Supervisor 重启后服务不自动启动

**解决**：
```bash
systemctl enable supervisor
# 确认配置中 autostart=true
```

### Q5: 如何查看定时任务是否正常执行

```bash
# 查看后端日志中的定时任务记录
grep "每日选股分析" /var/log/stock-strategy/backend.log | tail -5

# 或通过API查询
curl -s http://127.0.0.1:8000/api/system/status
```

### Q6: 需要更新代码怎么操作

```bash
# 方式1：Git pull + 重新部署
cd /opt/stock-strategy
git pull
./deploy.sh

# 方式2：重新上传后执行部署脚本
scp -r ./backend/* root@server:/opt/stock-strategy/backend/
scp -r ./frontend/* root@server:/opt/stock-strategy/frontend/
ssh root@server "/opt/stock-strategy/deploy.sh"
```
